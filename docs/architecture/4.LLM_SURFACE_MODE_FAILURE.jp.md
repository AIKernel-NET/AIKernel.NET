# 表面模写モードの危険性

表面模写モード（Surface Mode）は、LLM が **推論を停止し、表面構造の模倣に専念してしまう危険な状態**である。  
AIKernel のアーキテクチャは、この failure mode を構造的に防ぐために設計されている。

---

# 1. 表面模写モードとは何か

表面模写モードとは、LLM が **抽象推論を放棄し、入力の表面的特徴（文体・例・テンプレート）を模倣するだけの状態**である。

この状態では、LLM は「考える」ことをやめ、  
「見たものを真似る」だけの振る舞いになる。

---

# 2. 表面模写モードの特徴

アップロード文書の内容を基に整理すると、特徴は以下の通り：

- 文体だけ真似る  
- 推論しない  
- 構造を理解しない  
- 目的を見失う  
- 一瞬で“馬鹿”になる（＝推論能力の急激な低下）  
- **自己修正能力（Self-Correction）を失う**

### Self-Correction が失われる理由
表面模写モードでは、LLM は「構造」ではなく「見た目」を正解として扱うため、  
自分の誤りを検知するための **抽象的比較基準** を失う。

その結果：

- 誤りを誤りとして認識できない  
- 推論の軌道修正ができない  
- デバッグ不能状態に陥る  

これは AI システムとして致命的である。

---

# 3. なぜ表面模写モードが起こるのか（Attention 汚染との関係）

表面模写モードは **Attention 汚染の直接的な結果**である。

### 原因となるメカニズム
- 例やテンプレートは **トークン密度が高く、attention の重力が強い**  
- 抽象構造（structure）はトークン密度が低く、attention 競合に弱い  
- 結果として、LLM は **重力の強い表面構造に引き寄せられる**

### Few-shot 信仰の崩壊
従来の AI 開発では Few-shot（例示）は「精度を上げる魔法」とされてきた。  
しかし実際には：

- 例は最強の attention 汚染源  
- 推論を停止させる毒  
- 表面模写モードの最強トリガー  

AIKernel はこの誤解を構造的に否定し、  
**Few-shot を「推論破壊要因」として再定義する**。

---

# 4. トリガー（何が表面模写モードを引き起こすか）

アップロード文書に記載されている通り：

- 例（example）  
- 文体模倣指示  
- テンプレート  
- RAG の断片  

これらはすべて **表面構造**であり、  
attention を強く引き寄せる「重力源」として働く。

特に例（example）は最も強力なトリガーであり、  
LLM を即座に表面模写モードへ落とす。

---

# 5. Failure Mode としての深刻性

表面模写モードは単なる「品質低下」ではなく、  
**推論能力の喪失という致命的な failure mode** である。

深刻な理由：

- 推論が完全に停止する  
- 目的が attention に乗らず、タスクを遂行できない  
- RAG と混ざるとハルシネーションが増加  
- deterministic replay が破壊される  
- 例や文体の影響が支配的になり、制御不能になる  
- **自己修正能力（Self-Correction）が消失し、デバッグ不能状態に陥る**

AIKernel がこの failure mode を最重要視する理由はここにある。

---

# 6. 回避策（AIKernel のアーキテクチャによる防止）

アップロード文書の回避策を、AIKernel の OS 設計として再構成すると以下になる。

## 6.1 例を隔離（ExpressionContext）
- 例は推論に混ぜず、出力整形時のみ使用  
- 表面模写モードのトリガーを遮断  

## 6.2 推論層と表現層の分離
- OrchestrationContext（推論）  
- ExpressionContext（表現）  
- MaterialContext（素材）  

この 3 層分離により、  
表面構造が推論に混入することを構造的に防ぐ。

## 6.3 Attention 汚染防止
- 情報カテゴリ分離  
- RAG の素材化  
- 履歴の要約と分離  
- 抽象構造（structure）を最優先で attention に乗せる  

これらはすべて **表面模写モードを防ぐための OS レベルの仕組み**である。

---

# 7. AIKernel の役割  
### ― 表面模写という重力から知能を救い出す OS

表面構造は強い attention の重力を持つ。  
LLM は放っておくと、この重力井戸に引きずり込まれ、  
推論能力を失う。

AIKernel はこの重力井戸から **推論のコアを脱出させるための「脱出速度（escape velocity）」を与える OS** である。

- カテゴリ分離  
- コンテキスト隔離  
- 前処理中心主義  
- structure-first の設計  

これらはすべて、  
**LLM を表面模写モードから救い出し、抽象推論へ導くための OS 的機構**である。
