# Attention 汚染理論

LLM の推論能力は **attention の純度**に依存する。  
attention が表面構造に奪われると、LLM は抽象推論を行えなくなり、  
文体模倣・例模倣に偏った **表面模写モード**へと遷移する。

AIKernel のアーキテクチャは、この汚染を OS レベルで防ぐために設計されている。

---

# 1. Attention 汚染とは何か

attention 汚染とは、LLM の注意資源が **推論に必要な抽象情報ではなく、表面的な情報に吸われる現象**である。

汚染が発生すると：

- 抽象構造（structure）が attention に乗らない  
- 目的（purpose）が弱まり、推論の方向性が失われる  
- chain-of-thought が浅くなる  
- reasoning depth が消失する  

結果として、LLM は「考える」ことをやめ、  
「見たものを真似る」モードに落ちる。

---

# 2. Attention の「重力」モデル

表面構造（例・テンプレート・文体）は、  
抽象構造よりも **トークン密度が高く、attention を強く引き寄せる “重力” を持つ**。

### 表面構造の重力が強い理由
- トークンが密集している  
- パターンが明確で、モデルが学習しやすい  
- 形式的特徴が attention に強く作用する  

### 抽象構造は重力が弱い
- トークン密度が低い  
- 抽象概念は attention の競合に弱い  
- 表面構造に埋もれやすい  

### 結果
LLM は **重力の強い表面構造に引き寄せられ、抽象推論から脱落する**。

AIKernel の目的は、この重力井戸から **推論のコアを脱出させる**ことである。

---

# 3. Attention の SNR（Signal-to-Noise Ratio）

attention の純度は、以下の比率で定義できる：

```
SNR = 推論信号（Signal） / 表面ノイズ（Noise）
```

### Signal（推論信号）
- 目的（purpose）
- 制約（constraints）
- 抽象構造（structure）
- 思考パターン（reasoning pattern）

### Noise（表面ノイズ）
- 例（example）
- 文体指示
- RAG の断片
- 履歴の混入
- ノイズ情報

SNR が低下すると：

- 推論が浅くなる  
- 表面模写モードに遷移  
- ハルシネーションが増加  

AIKernel のアーキテクチャは、  
**Signal を強化し、Noise を隔離することで SNR を最大化する OS 設計**である。

---

# 4. 汚染要因

アップロード文書に記載されている通り、attention を汚染する主な要因は以下である：

- 例（example / sample）
- 文体模倣指示
- RAG の断片
- 履歴の混入
- ノイズ情報

これらはすべて **表面構造**であり、  
推論に必要な抽象情報とは異なる。

---

# 5. 汚染が引き起こす Failure Mode

attention 汚染が発生すると、以下の failure mode が誘発される。

## 5.1 推論停止
- structure が attention に乗らない  
- 目的が弱まり、推論の方向性が消失  
- chain-of-thought が浅くなる  

## 5.2 表面模写モード
特徴：

- 文体だけ真似る  
- 推論しない  
- 構造を理解しない  
- 目的を見失う  

## 5.3 ハルシネーション増加
- RAG と context が混ざると事実性が崩壊  
- attention が「どこまでが事実か」を判断できなくなる  

## 5.4 目的の喪失
- purpose が attention に乗らず、  
  LLM が「何を達成すべきか」を理解できなくなる  

---

# 6. 理論の射程  
### ― モデルが賢くなっても解決しない問題

Attention 汚染は **モデルの性能とは無関係**である。  
これは「モデルが賢くなれば解決する問題」ではなく、  
**トークンベースのアーキテクチャである限り必ず発生する構造的問題**である。

理由：

- attention は有限資源である  
- トークン密度の高い表面構造は常に重力が強い  
- Signal と Noise の競合はモデルサイズに依存しない  
- SNR の低下はモデルの能力とは無関係に発生する  

したがって、  
**Attention 汚染は “モデルの限界” ではなく “情報構造の問題”** である。

AIKernel はこの問題を **OS レベルの構造**で解決する。

---

# 7. 防止策（AIKernel のアーキテクチャ）

アップロード文書に記載されている防止策を、  
AIKernel の OS 設計として再構成すると以下になる。

## 7.1 情報カテゴリ分離
- purpose / constraints / structure / expression / rag_material / history を分離  
- 混在を構造的に禁止  

（参照：1.CATEGORY_SEPARATION_PRINCIPLES.jp.md）

## 7.2 例の隔離（ExpressionContext）
- 例は推論に混ぜず、出力整形時のみ使用  
- 表面模写モードのトリガーを遮断  

## 7.3 RAG の素材化（MaterialContext）
- RAG を生データのまま渡さない  
- 必ず構造化してから推論に転写  

## 7.4 推論層と表現層の分離
- OrchestrationContext（推論）  
- ExpressionContext（表現）  
- MaterialContext（素材）  

この 3 層分離により、  
attention 汚染が構造的に防止される。

---

# 8. AIKernel における位置づけ

Attention 汚染理論は、AIKernel の以下の設計原則の基盤となっている：

- 情報カテゴリ分離  
- コンテキスト隔離  
- 表面模写モード防止  
- 前処理中心主義（Preprocessing over Prompting）  
- deterministic replay の保証  

AIKernel のアーキテクチャは、  
**attention の純度を守るための OS 的構造**として設計されている。
