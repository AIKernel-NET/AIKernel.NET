# Capability Definition Schema
`ModelCapacityVector` と動的ルーティングで使用する能力次元（Dimensions）、スコアリング基準、および Provider の宣言スキーマを定義する。

## 1. Purpose
AIKernel においてモデルは交換可能コンポーネントであり、選定は推測ではなく計算で行うべきである。本仕様は次を実現する。

- Provider 能力宣言の標準化: 異なるベンダー能力を共通尺度で比較可能にする。
- 監査可能なモデル選定: 要求ベクトルと能力ベクトルの適合をスコアで説明可能にする。
- 能力ベース抽象化: モデル名ではなく能力要件（例: `Logic_Depth >= 0.8`）で指定可能にする。

## 2. Core Dimensions (能力次元)
各次元は `0.0..1.0` でモデル特性を表現する。

| 次元名 | 定義 | 評価の主眼 |
|---|---|---|
| `Logic_Depth` | 多段推論・数学・論理 | 推論手順を崩さず結論へ到達できるか |
| `Creative_Fluency` | 多様性・創造性 | 表現の豊かさと指示範囲内の創造品質 |
| `Constraint_Rigidity` | 規則準拠・形式 | JSON Schema、禁止事項、システム指示順守 |
| `Token_Economic_Efficiency` | 費用対効果 | 低コストで達成できる推論品質 |
| `Latency_Stability` | 応答安定性 | TTFT とスループットの安定性 |
| `Tool_Use_Reliability` | ツール呼び出し決定性 | ツール選択精度と引数ハルシネーション抑制 |

## 3. Scoring and Calibration (スコアリングと較正)
スコアは自己申告ではなく証拠駆動で評価する。

- Scoring Scale: `0.0`（能力なし）から `1.0`（最高水準）。
- Evidence Sources:
  - 公開ベンチマークを AIKernel 評価器で再計算した値
  - `Deterministic Replay` ログに基づく実タスク成功率
  - ガバナンスインシデント（制約違反等）による減点
- Calibration Metadata（必須）:
  - `EvaluationDate`
  - `EvaluatorVersion`
  - `ConfidenceScore`

## 4. Declaration Contract
Provider 実装は `IKernelHost` 登録時に次を公開する。

- Static Capability Profile: モデル固有の準不変能力値
- `IDynamicCapacityProvider`: 実行時負荷・予算でベクトル補正する動的成分
- `IExecutionConstraints`: 最大トークン、対応 MIME などの技術制約

## 5. Validation Rules (検証規則)
AIKernel は宣言値を検証し、次の条件で拒否または劣化扱いにする。

1. 次元網羅性: 必須次元欠落 Provider は登録拒否。
2. 劣化プロファイル: 較正期限超過（例: 90 日）スコアは減衰または非信頼扱い。
3. 単位整合性: ベンチ基準が Schema Version と不一致なら登録拒否。

## 6. Fail-Closed Integration
`IModelVectorRouter` は次の条件で候補 Provider を即時除外する。

- Schema Mismatch: 能力宣言スキーマが要求バージョン未満。
- Constraint Violation: Mandatory 軸閾値を満たす候補が存在しない。
- Low Trust Level: 能力証拠が非公式/無署名等で `TrustPolicy` 閾値未満。
