# Capability Definition Schema
Defines capability dimensions, scoring criteria, and provider declaration schema used by `ModelCapacityVector` and dynamic routing.

## 1. Purpose
In AIKernel, models are replaceable components. Selection must be computed, not guessed. This schema provides:

- Standardized provider capability declaration across vendors.
- Auditable model selection via requirement/capability fit scores.
- Capability-based abstraction where developers specify requirements (for example `Logic_Depth >= 0.8`) instead of model names.

## 2. Core Dimensions
Each dimension expresses model characteristics on a `0.0..1.0` scale.

| Dimension | Definition | Evaluation Focus |
|---|---|---|
| `Logic_Depth` | Multi-step reasoning, math, logic | Ability to preserve reasoning chain to conclusion |
| `Creative_Fluency` | Diversity and creativity | Expressive richness within instruction bounds |
| `Constraint_Rigidity` | Rule and format compliance | JSON Schema, forbidden-rule, and system-priority compliance |
| `Token_Economic_Efficiency` | Cost efficiency | Reasoning quality achieved per cost/token unit |
| `Latency_Stability` | Response stability | TTFT and throughput consistency |
| `Tool_Use_Reliability` | Tool-call determinism | Accuracy of tool selection and low argument hallucination |

## 3. Scoring and Calibration
Scores must be evidence-driven, not self-claimed.

- Scoring Scale: `0.0` (none) to `1.0` (top tier).
- Evidence Sources:
  - public benchmark results recomputed by AIKernel evaluators
  - real-task success rates from `Deterministic Replay` logs
  - negative adjustments from governance incidents
- Required Calibration Metadata:
  - `EvaluationDate`
  - `EvaluatorVersion`
  - `ConfidenceScore`

## 4. Declaration Contract
When a provider is registered via `IKernelHost`, it must publish:

- Static Capability Profile: semi-stable model-intrinsic capability values
- `IDynamicCapacityProvider`: runtime correction component for load/budget conditions
- `IExecutionConstraints`: hard limits (max tokens, supported MIME types, etc.)

## 5. Validation Rules
AIKernel validates declarations and rejects/degrades when:

1. Coverage violation: missing mandatory dimensions causes registration reject.
2. Degraded profile: stale calibration window (for example 90 days) applies decay or marks profile untrusted.
3. Unit inconsistency: benchmark basis mismatching schema version causes reject.

## 6. Fail-Closed Integration
`IModelVectorRouter` must immediately exclude provider candidates when:

- Schema Mismatch: declared capability schema is older than required version.
- Constraint Violation: no candidate satisfies mandatory-axis thresholds.
- Low Trust Level: evidence sources are unofficial/unsigned and below `TrustPolicy` threshold.
