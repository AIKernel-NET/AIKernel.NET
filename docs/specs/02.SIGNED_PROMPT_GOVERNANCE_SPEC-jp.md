# 02. Signed Prompt Governance Spec

## 1. Purpose
署名済みプロンプトのみを実行可能にする Fail-Closed 統治契約を定義する。

## 2. Scope
- `IPromptRepository`
- `IPromptVerifier`
- `IPromptSignatureProvider`
- `ISignatureTrustStore`
- `IPromptValidator`

## 3. Normative Requirements
- `SGS-001` 署名欠落アーティファクトは拒否しなければならない（MUST）。
- `SGS-002` `IPromptVerifier` はハッシュ整合性を検証しなければならない（MUST）。
- `SGS-003` `ISignatureTrustStore` は署名者信頼/失効を判定しなければならない（MUST）。
- `SGS-004` `IPromptValidator` は有効期限とスコープを検証しなければならない（MUST）。
- `SGS-005` 署名対象の正規化（Canonical Signing Payload）: ROM の `entity.id`、`version`、`policy`、本文（Prompt Body）を正規化した意味的スナップショットに対して署名しなければならない（MUST）。
- `SGS-006` 信頼連鎖の検証（Validation of Trust Chain）: `ISignatureTrustStore` は署名鍵が正当 issuer により承認され、実行時点で失効していないことを証明しなければならない（MUST）。
- `SGS-007` 実行スコープの動的束縛（Dynamic Scope Binding）: `IPromptValidator` は `allowed_tools`、`max_token_budget`、`scopes` が実行直前コンテキスト（権限/予算）内にあることを検証しなければならない（MUST）。

## 4. Fail-Closed
- `SGS-F001` 判定不能（Indeterminate）は `Deny` とする。
- `SGS-F002` 検証基盤障害時は実行停止する。
- `SGS-F003` フォールバック実行モードを禁止する。
- `SGS-F004` Temporal Integrity Violation: `created_at` が未来時刻、または `expires_at` 超過時は即時ロード拒否する。
- `SGS-F005` Trusted Anchor Unreachability: `ISignatureTrustStore` 到達不能（タイムアウト/通信障害）時は `Indeterminate` として実行遮断する。

## 5. Acceptance Criteria
- AC1: 未署名で実行不可。
- AC2: 失効署名者で実行不可。
- AC3: 正常署名+正当スコープでのみ実行可能。
- AC4: 期限超過アーティファクトは 1 秒超過でも拒否される。
- AC5: TrustStore 到達不能時は常に `Deny` となる。

## 6. Artifact Anatomy（ROM/YAML 構造）
署名済みプロンプトは以下形式で `IPromptRepository` に保存される。

```yaml
---
# Identity & Logic
id: "logic.analyser.core"
version: "2.1.0"
type: "kernel.signed_prompt"

# Policy & Constraints
policy:
  trust_level_required: 0.9
  allowed_providers: ["[[provider.secure_llm]]"]
  scopes: ["read:financial_data", "write:audit_log"]

# Governance & Signature
governance:
  issuer: "ai-kernel@tkysoftware.xsrv.jp"
  signer_id: "gov-officer-01"
  hash_alg: "SHA256"
  hash: "sha256:7f8e9a1b..."
  signature: "MEYCIQ... (ECDSA/RSA Signature)"
---
# Prompt Body (ROM Format)
## Task: Analyze Financial Statements
* [constraint]: Do not disclose PII.
* [reference]: [[material.latest_report]]
```

## 7. Governance Workflow（統治シーケンス）
`IPromptVerifier` は次の手順で正当性を判定する。

1. Format Validation  
   YAML 必須項目（`id`, `hash`, `signature`）の存在を確認する。
2. Integrity Check  
   ROM 本文+メタデータからハッシュを再計算し `hash` と一致確認する。
3. Signature Authentication  
   `IPromptSignatureProvider` で署名が正当秘密鍵由来か検証する。
4. Trust Resolution  
   `ISignatureTrustStore` で `signer_id` 信頼状態と鍵失効状態を確認する。
5. Policy Authorization  
   `IPromptValidator` で policy が現行実行制約内か検証する。
6. Unfreezing  
   すべて通過時のみ `Orchestration` バッファへ展開し実行許可する。
