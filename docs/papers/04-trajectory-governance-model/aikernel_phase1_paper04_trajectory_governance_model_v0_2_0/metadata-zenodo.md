## Description
This technical note is Part II-4 of the AIKernel / AIOS Phase-1 specification series. It defines Trajectory Governance as a deterministic governance model for evaluating semantic state transitions, candidate actions, convergence, goal alignment, risk, uncertainty, and repetition after inference has begun.

The paper treats LLM / SLM outputs as stochastic proposals rather than directly executable system transitions. Candidate transitions are evaluated by a deterministic Policy Decision Point and mapped to controlled governance decisions such as Permit, Deny, AskConfirmation, Clarify, or Abort.

This v0.2.0 series edition is adapted from the standalone archived version of the Trajectory Governance Model, DOI: 10.5281/zenodo.20223205.

The English manuscript is the canonical version. The Japanese manuscript is included as a companion translation.
